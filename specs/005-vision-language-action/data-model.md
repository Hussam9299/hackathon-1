# Data Model for Module 4: Vision-Language-Action (VLA)

This document outlines the key conceptual data entities for the VLA module.

## 1. Natural Language Command
A user's instruction to the robot, provided as a text string.
- **Attributes**:
    - `text`: (string) The raw text of the command (e.g., "bring me the water bottle").
    - `timestamp`: (datetime) When the command was issued.

## 2. Cognitive Plan
A high-level, multi-step plan generated by an LLM to fulfill the user's command.
- **Attributes**:
    - `steps`: (list of strings) An ordered list of actions in natural language (e.g., ["1. Navigate to the kitchen", "2. Find the water bottle", "3. Pick it up"]).
- **Relationships**:
    - Generated from a `Natural Language Command`.
    - Translated into a `ROS 2 Action Sequence`.

## 3. ROS 2 Action Sequence
A sequence of concrete, executable actions for the robot's controllers.
- **Attributes**:
    - `actions`: (list of structs) An ordered list of ROS 2 actions or service calls, including parameters (e.g., `[{action: '/navigate_to_pose', goal: {x: 1.0, y: 2.5}}, {action: '/pick_object', goal: {object_id: 'water_bottle'}}]`).
- **Relationships**:
    - Generated from a `Cognitive Plan`.

## 4. VLA Pipeline
The integrated system that manages the flow from command to execution. This is a system entity rather than a data entity.
- **Components**:
    - Speech-to-Text Engine
    - LLM Planner
    - Perception System
    - Navigation System
    - Manipulation System
