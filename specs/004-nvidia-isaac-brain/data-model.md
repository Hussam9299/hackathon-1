# Data Model for Module 3

This document outlines the key data entities relevant to the "AI-Robot Brain (NVIDIA Isaac™)" module. As this is a documentation and learning module, these are conceptual entities rather than database models.

## 1. Docusaurus Chapter
Represents a single learning section within the Docusaurus book.
- **Attributes**:
    - `title`: (string) The title of the chapter.
    - `content`: (string) The body of the chapter in MDX format.
    - `order`: (integer) The position of the chapter in the sidebar.
- **Relationships**:
    - Belongs to a single Module.

## 2. Isaac Sim Environment
A simulated 3D world used for training and testing the robot.
- **Attributes**:
    - `name`: (string) A unique name for the environment.
    - `scene_description`: (text) Description of the objects and layout of the environment.
    - `robot_model`: (file) The URDF or other model file for the robot being simulated.
- **Relationships**:
    - Contains multiple `Synthetic Sensor Data` streams.

## 3. Synthetic Sensor Data
Data generated by virtual sensors within the Isaac Sim Environment.
- **Attributes**:
    - `sensor_type`: (enum) The type of sensor (e.g., 'Camera', 'LIDAR', 'IMU').
    - `data_format`: (string) The format of the data (e.g., 'jpeg', 'pointcloud2').
    - `timestamp`: (datetime) The simulation time when the data was generated.
    - `data`: The actual sensor data.
- **Relationships**:
    - Generated by a sensor within an `Isaac Sim Environment`.
    - Consumed by an `Isaac ROS Pipeline`.

## 4. Isaac ROS Pipeline
A computational graph for processing sensor data.
- **Attributes**:
    - `name`: (string) The name of the pipeline (e.g., 'VSLAM', 'Object Detection').
    - `input_topic`: (string) The ROS 2 topic the pipeline subscribes to for sensor data.
    - `output_topic`: (string) The ROS 2 topic the pipeline publishes results to.
    - `nodes`: (list) The list of ROS 2 nodes that make up the pipeline.
- **Relationships**:
    - Consumes `Synthetic Sensor Data`.
    - Produces data that can be used by the Nav2 stack.

## 5. Nav2 Goal
A target destination for the robot within the simulated environment.
- **Attributes**:
    - `pose`: (struct) The target position (x, y, z) and orientation (quaternion) of the robot.
- **Relationships**:
    - Sent to the Nav2 stack to initiate navigation.
