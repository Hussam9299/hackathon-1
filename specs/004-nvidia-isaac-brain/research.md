# Research for Module 3: The AI-Robot Brain (NVIDIA Isaac™)

This document outlines the decisions made to resolve the "NEEDS CLARIFICATION" items from the implementation plan.

## 1. Technology Versions

### Decision
- **NVIDIA Isaac Sim**: Use version 5.1.0 or later. This version is open source, providing greater flexibility.
- **Isaac ROS**: The packages are compatible with ROS 2 Humble. We will use the latest available versions for Humble.
- **Nav2**: A stable and maintained version is available for ROS 2 Humble. We will use the latest available version for Humble.

### Rationale
- Recent searches confirm that Isaac Sim 5.1.0 is the latest version with significant improvements and is now open source.
- Isaac ROS and Nav2 are both actively maintained for ROS 2 Humble, which is a target platform for this project as per `GEMINI.md`. Using the latest stable versions ensures access to the latest features and bug fixes.

## 2. Docusaurus Configuration

### Decision
- **Docusaurus Version**: Continue using Docusaurus v3, as it is already set up in the project.
- **Preset**: Continue using the `'classic'` preset.

### Rationale
- The existing `book/` directory is a Docusaurus v3 site using the classic preset. Maintaining this consistency avoids unnecessary refactoring and configuration changes.

## 3. Module and Chapter Structure

### Decision
The module will be broken down into three chapters:
1.  **Chapter 1: Introduction to Isaac Sim for Robotics**: Covers setting up scenes, importing robots, and the basics of synthetic data generation.
2.  **Chapter 2: Hardware-Accelerated Perception with Isaac ROS**: Focuses on using Isaac ROS pipelines for tasks like VSLAM.
3.  **Chapter 3: Navigation with Nav2**: Introduces the core concepts of the Nav2 stack and demonstrates a basic navigation workflow.

### Rationale
- This structure provides a logical progression from simulation environment setup to perception and finally to action (navigation), aligning with the user stories in the feature specification. It fits within the 2-3 chapter constraint.

## 4. Sidebar and Navigation

### Decision
- The sidebar will be auto-generated based on the directory structure, as configured in `sidebars.ts`.
- The new module will be placed in `book/docs/module3-nvidia-isaac/`, and the chapters will be named to ensure correct alphabetical and logical ordering (e.g., `1-introduction.md`, `2-perception.md`, `3-navigation.md`).

### Rationale
- The current `sidebars.ts` uses autogeneration (`[{type: 'autogenerated', dirName: '.'}]`), which is simple and maintainable. Adhering to this convention is the most straightforward approach.

## 5. Content Standards

### Decision
- **Markdown**: All content will be written in Docusaurus-flavored Markdown (MDX), which allows for React components within Markdown.
- **Code Blocks**: All code blocks must include a language specifier for correct syntax highlighting (e.g., ` ```python `).
- **Diagrams**: All diagrams must be in SVG format and stored in the `static/img` directory.

### Rationale
- These standards are consistent with Docusaurus best practices and the existing project constraints.

## 6. Deployment Strategy

### Decision
- Deployment will be handled via GitHub Pages.
- A GitHub Actions workflow will be created to automate the build and deployment process. The workflow will trigger on pushes to the `main` branch.
- The workflow will use the `npm run deploy` command provided by Docusaurus.

### Rationale
- The project constitution specifies GitHub Pages deployment. Using GitHub Actions for CI/CD is a standard and efficient way to automate this process. Docusaurus provides a simple deployment command that integrates well with this setup.
